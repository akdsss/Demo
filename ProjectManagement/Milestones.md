# Milestones

## M0 项目骨架

验收标准：

- Godot C# 项目可打开并启动。
- 推荐目录结构已建立。
- 能进入一个空白启动场景或战斗测试场景。
- 数据目录存在，并有最小示例数据。
- UTF-8 without BOM 数据检查方案已记录。

## M1 战斗结算内核

验收标准：

- 支持 1 到 6 号时间点。
- 能创建 `PlannedAction` 并放入时间轴。
- 技能合法性检查覆盖 GDD 中的基础失败原因。
- 支持移动、近战、远程、防御、治疗的基础效果。
- 支持技能标签：近战、远程、移动、防御、单体、范围、治疗。
- 支持基础区域修正与状态触发。
- 结算输出结构化 `CombatEvent`。
- 可通过测试或调试场景验证一次完整回合。

## M2 战斗 UI 原型

验收标准：

- 显示我方与敌方时间轴。
- 显示战斗区域和角色站位。
- 显示“设置指令”“检视详情”“开始结算”。
- 鼠标悬停技能时显示详情。
- 长按 2 秒可在空白时点放置技能，并显示进度条。
- 指令确认后不可撤回。
- 已设置指令的时点不可覆盖。
- 左键确定、右键返回生效。
- 开始结算后能播放战斗记录。

## M3 教学关可玩

验收标准：

- 第一关可从入口进入。
- 教学步骤按顺序触发，能引导玩家完成一次完整回合。
- 教学解释时间轴、区域修正、技能放置、怪物意图揭示、百科和演出阶段。
- 百科入口可打开，至少包含区域修正和状态效果条目。
- 基础敌人 AI 能生成行动。
- 玩家能胜利或失败，并进入关卡结算。

推荐教程步骤：

1. 开场说明：战斗不看距离，只看结算时敌我所在区域和时间点。
2. 时间轴说明：每回合有 6 个时点，每个时点只能放一个指令。
3. 区域说明：八卦区域会对伤害、治疗、状态和优先级产生修正。
4. 百科引导：提示右上角书本入口，要求玩家打开百科并查看一个区域条目。
5. 设置指令：引导玩家选择近战角色的移动或近战技能。
6. 放置指令：要求玩家在空白时点长按 2 秒确认指令。
7. 检视详情：引导玩家查看一个已放置指令；未揭示敌方行动只显示标签。
8. 怪物行动揭示：当所有行动机会用完后，揭示怪物行动，解释优先级和技能名称。
9. 开始结算：要求玩家点击“开始结算”，进入演出阶段。
10. 战斗记录说明：展示行动、伤害、治疗、战败等日志信息。
11. 区域修正示例：安排一次移动，让玩家看到离开/进入区域后修正变化。
12. 胜利结算：击败教学敌人后展示属性成长或新技能提示。

## M4 三关 Demo

验收标准：

- 第二关小型敌方团体可玩。
- 第三关双精英战可玩。
- 狂怒状态能按 GDD 触发并改变行动次数/伤害。
- 支援敌人能治疗、远程攻击和释放延迟范围伤害。
- 关卡胜利后展示三名角色头像、属性变化和新技能。
- 已解锁技能能在后续关卡使用。
- 三关均可连续游玩。

## M5 打磨与资源替换

验收标准：

- 静态美术资源统一水墨风。
- UI 图标、角色头像、敌人头像、区域图标齐全。
- 战斗播片占位资源路径完整。
- UI hover/click、阶段提示、行动确认音效接口可用。
- 百科内容可滚动或分页。
- 关键规则有测试或调试验证。
- 无明显中文乱码、缺图、缺配置、阻塞性错误。
