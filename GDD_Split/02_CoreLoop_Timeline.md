# 核心循环与时间轴

## 核心体验

玩家在准备阶段把角色指令放入时间轴，规划当前回合的行动。当场上无可行动角色时，将所有指令按时间点与优先级预先在后台结算，并预先确定战斗记录内容、战斗播片内容。玩家点击“开始结算”并确定后，进入演出阶段，此时仅仅显示战斗记录与播放战斗播片。

## 回合结构

每回合分为两个阶段：

- 准备阶段：玩家和敌人生成或放置本回合行动。当场上无可行动角色时，根据已确定的行动计划执行结算。
- 演出阶段：播放动画、显示战斗记录。

## 时间轴

当前可辨认设计为六个时点。每个时点只能放置一个指令。玩家侧和敌方侧各有自己的时间轴。

建议实现：

- `Timeline`：保存 1 到 6 号槽。
- `TimelineSlot`：保存时点、阵营、行动、锁定状态、预览信息。
- `PlannedAction`：保存释放者、技能、目标、目标区域、释放时点、持续/延迟信息。

## 准备阶段流程

1. 显示“第 N 回合”。
2. 播放对应阶段提示音效，约 1.5 秒后显示“准备阶段”。
3. 怪物优先准备。时间轴右侧显示怪物“还剩 X 次”，计算时显示“思考中”。
4. 每个仍有行动次数的怪物在时间轴上放置一次行动；怪物行动放置同时完成，并有音效提示。
5. 玩家开始准备，左侧显示战斗 UI 交互控件。
6. 玩家对拥有剩余行动次数的每个角色安排一次行动，于时间轴对应位置长按以确定。
7. 指令确定后有音效提示，且不可撤回。
8. 可行动角色全部安排完成后，自动进入怪物的下一轮准备。
9. 场上无可行动角色后，立刻揭示所有怪物行动，在后台开始时间轴结算，并在左侧出现“开始结算”按钮。

## 敌方行动生成

敌方行动建议在准备阶段开始时就后台生成，但 UI 可逐步表现“思考中”。

注意：

- 敌方行动生成不应依赖玩家后续放置的行动，除非设计明确允许敌人响应玩家。
- 若需要让敌人受玩家已放置行动影响，应在文档中明确“敌人行动生成时机”。

## 演出阶段流程

1. 锁定双方时间轴。
2. 读取准备阶段后台已经完成的 `CombatEvent` 列表。
3. UI 按事件顺序播放战斗播片、数字、状态变化和日志。
4. 回合结束效果结算。
5. 检查胜负。
6. 进入下一回合或关卡结算。

## 实现建议

不要在演出动画中做真实结算。真实结算应一次性完成并产出事件列表。这样可以支持：

- 战斗记录回放。
- 快进或跳过动画。
- 自动测试结算结果。
- Debug 对比“计划行动”和“实际事件”。
- 用占位播片资源先搭通流程，后续替换正式战斗播片。

## 最小可行版本

MVP 只需支持：

- 六时点槽位。
- 玩家选择角色、技能、目标、时点。
- 敌人随机或简单规则放置行动。
- 按时点和优先级执行。
- 输出战斗日志。
