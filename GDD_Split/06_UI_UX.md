UI 与交互

# 战斗主界面

建议主要区域：

- 上方：敌方时间轴。
- 中部：八卦区域战场。
- 下方：玩家时间轴。
- 左侧：战斗UI交互控件
- 右侧：当前角色/技能/战斗记录，根据阶段切换。

全局输入约定：

- 左键：确定。
- 右键：返回。

# 区域显示与区域目标选择

未来我阅读此文档时必须把区域 UI 理解为显示层，不是结算层。

- 战斗地图背景仍显示十区域：中间太极阴阳，外环八卦。
- 每个区域配置 9 个固定透明头像锚点；锚点透明度 100%，不显示格子，不作为按钮。
- 锚点只负责角色头像显示位置。角色进入/离开区域时，只移动该角色，不重排该区域其他角色。
- 角色当前区域由数据字段 `CurrentAreaId` 决定。头像所在锚点必须跟随 `CurrentAreaId`，但范围伤害不依赖头像坐标。
- 玩家选择目标区域时不点击地图、不点击区域、不点击锚点。
- 玩家选择需要区域目标的技能后，打开模态次级菜单；菜单显示 10 个矩形按钮，每个按钮是一个区域名。
- 点击某个区域矩形后，写入 `TargetAreaId` 并返回后续指令放置流程。
- 点击 10 个区域矩形之外的地方，只关闭次级菜单并回到上一级菜单。
- 区域次级菜单打开时，菜单外点击必须吞掉输入，不允许穿透触发地图、时间轴、头像、技能按钮或其他左键事件。
- 区域菜单是选择 `TargetAreaId` 的唯一 UI 入口。地图热点或碰撞体不是当前方案的一部分。

# 准备阶段 UI

## 战斗UI交互控件

- 显示在屏幕左侧，我方时间轴的上方区域。只在准备阶段且我方行动时显示出。
- 包含“设置指令”、“检视详情”、“开始结算”
- **指令提交后重置指令选择上下文**，包含隐藏技能菜单、清空当前技能/目标/区域选择、退出时间轴放置模式，并让下一次点击“设置指令”重新解析当前仍有行动次数的角色。

### 设置指令

1. 点击“设置指令”，进入次级菜单，显示我方角色。其中战败或行动次数为0的显示灰色且不可选区。
2. 点击后从上往下依次出现“近战”“位移”“远程”“特殊”（若该角色无某一类型技能则不显示）。
3. 点击后展示拥有对应标签的技能。
4. 用户鼠标置于技能上时，对应技能字体放大，并在技能列表下方展示技能详情，包括优先级、MP消耗与技能效果介绍。
5. 单击技能后，于时间轴上空白节点长按1.2秒以放置技能，需要做一个随长按时间增长的进度条效果。并在我方时间轴上方“我方行动”的上方显示“于时间轴上空白节点长按以放置技能”
6. 在选择 以角色为目标的技能 后，应该展示一个次级列表，包含所有敌人名称，其中已战败的敌人显示为灰色，不可选取。

### 检视详情

点击后，鼠标置于非空白时点上，在右侧界面显示该技能的描述。若未被揭示，仅显示其标签。若已被揭示，显示其优先级、描述、目标区域/角色（若该技能需要指定）。

### 开始结算

当场上所有单位均无可行动机会，将“设置指令”替换为“开始结算”。

点击后弹窗“是否进入演出阶段”，点击“是”进入演出阶段，点击“否”取消弹窗。

## 敌方UI

- 位于屏幕上侧区域。
- 对于一位具体的敌人，中间为时间轴，是长宽比很大的长矩形，分为六个区域。时间轴同行的左侧，从左往右依次显示敌方头像（素材说明详见PictureAsset.md）、生命值（按照“当前生命值/最大生命值”的格式）、敌方名称。时间轴同行的右侧，显示怪物剩余行动次数（按照”剩余X次“的格式，X为剩余行动次数）（若X=0，显示”完成“）。
- 每名敌人都配置这样一个UI，按照EnemyDesign.md出现顺序从上往下排列。被击败后UI依旧保留。
- 在时间轴上显示的敌方技能默认为未揭露状态，只显示技能标签。当场上无可行动角色后，揭示所有怪物行动，揭示后显示优先级、描述、目标区域或角色。（战斗中，我方支援角色可使用揭露技能）

## 我方UI

- 位于屏幕下侧区域。
- 对于一位具体的我方角色，中间为时间轴，是长宽比很大的长矩形，分为六个区域。时间轴的各个尺寸要求与敌方UI一致。时间轴同行的左侧，从左往右依次显示我方头像、生命值（按照“当前生命值/最大生命值”的格式）、MP（按照”当前MP/最大MP）、我方角色名称。
- 每名我方角色都配置这样一个UI，按照PlayerDesign.md出现顺序从上往下排列。被击败后UI依旧保留。
- 准备阶段，当轮到玩家行动时，在我方UI的最上面的时间轴的上方，显示一个长度为屏幕长度，宽度很小的矩形，内有“我方行动”居中文本，矩形颜色为浅绿，文本为黑色加粗。（其余情况隐藏这个UI）

（我方UI与敌方UI总体逻辑几乎一致，只多一个MP显示与“我方行动”显示）

## 交互建议：

- 选择技能后，高亮合法目标和合法时间点。
- 鼠标悬停技能时显示简短说明。
- 鼠标悬停技能时，对应技能字体放大，并在技能列表下方展示优先级、MP 消耗与效果介绍。
- 放置指令前必须经过统一合法性检查。
- 时间轴空白节点需要长按 2 秒放置技能，并显示随长按增长的进度条。
- 已确认的指令不可撤回。
- 已设置指令的时点暂不允许覆盖。

# 演出阶段 UI

- 如上所述，时间轴的结算应该于场上无可行动角色后立刻在后台完成。而在演出阶段，只是根据结果，依次于右侧显示具体战斗记录，于整个画面播放对应的播片。
- 战斗记录需要显示：（头像）（名称）（做出何种行动）。若该行动造成伤害，显示（头像）（名称）（受到的具体伤害数值）。当有角色生命值归零时，显示（头像）（名称）（战败）

GDD 当前说明攻击、受击等战斗相关音效不需要考虑，因为战斗以播片方式演出。但控件 hover/click、阶段提示、行动确认等 UI 音效仍可保留接口。

# 战斗记录

建议使用结构化日志：

- 头像。
- 名称。
- 行动名称。
- 目标。
- 伤害/治疗数值。
- 技能释放失败原因。
- 状态变化。

不要直接在结算代码里拼 UI 文本。

百科界面要求：

- 左侧区域以列表形式展示条目合集与下属条目。
- 右侧区域呈现当前选中条目的详细内容。
- 初始状态下右侧内容为空。
- 内容高度超出显示区域时，应支持垂直滚动；若实现滚动困难，可拆成分页。

# 教程

教程建议采用脚本化步骤：

- 提示文本。
- 高亮 UI 节点。
- 等待玩家操作。
- 限制可点击区域。
- 完成后进入下一步。

这样便于把教学关和正常战斗复用同一套 UI。

# 成长/奖励界面

关卡胜利后显示：

- 角色头像。
- 属性变化。
- 新技能或技能升级。
- 继续按钮。

建议这个界面只读取 `growth_rewards` 配置。
