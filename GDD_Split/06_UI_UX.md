# UI 与交互

## 战斗主界面

建议主要区域：

- 上方：敌方时间轴。
- 中部：八卦区域战场。
- 下方：玩家时间轴。
- 左侧：当前角色/技能/战斗记录，根据阶段切换。
- 右侧：区域修正、敌人列表或百科入口。

全局输入约定：

- 左键：确定。
- 右键：返回。

## 准备阶段 UI

可用控件：

- 指令分类按钮：近战、位移、远程、特殊等。
- 技能列表。
- 技能详情：优先级、MP、目标类型、效果说明。
- 时间轴槽位。
- 撤销或取消当前选择。
- 开始结算按钮。

交互建议：

- 选择技能后，高亮合法目标和合法时间点。
- 鼠标悬停技能时显示简短说明。
- 鼠标悬停技能时，对应技能字体放大，并在技能列表下方展示优先级、MP 消耗与效果介绍。
- 放置指令前必须经过统一合法性检查。
- 时间轴空白节点需要长按 2 秒放置技能，并显示随长按增长的进度条。
- 已确认的指令不可撤回。
- 已设置指令的时点暂不允许覆盖。

## 敌方行动显示

文档中有“思考中”和敌方行动生成表现。建议：

- 敌方行动真实数据在后台生成。
- UI 可延迟显示，用于表现敌人思考。
- 不要让 UI 表演改变 AI 结果。
- 场上无可行动角色后，揭示所有怪物行动；未揭示前，检视详情只显示技能标签，揭示后显示优先级、描述、目标区域或角色。

## 演出阶段 UI

演出阶段只播放结算事件：

- 技能名与释放者。
- 约 3 秒战斗播片；当前阶段使用占位资源搭建。
- 角色移动或位置变化。
- 伤害/治疗飘字。
- 状态图标添加/移除。
- 战败提示。
- 战斗记录滚动。

建议提供：

- 快进。
- 跳过演出。
- 暂停查看日志。

GDD 当前说明攻击、受击等战斗相关音效不需要考虑，因为战斗以播片方式演出。但控件 hover/click、阶段提示、行动确认等 UI 音效仍可保留接口。

## 战斗记录

建议使用结构化日志：

- 头像。
- 名称。
- 行动名称。
- 目标。
- 伤害/治疗数值。
- 失败原因。
- 状态变化。

不要直接在结算代码里拼 UI 文本。

## 百科

入口位于右上角，建议使用书本类外观。内容包括：

- 区域条目。
- 状态效果条目。
- 技能规则条目。
- 基础战斗规则。

建议百科从数据配置生成基础描述，额外长文用独立字段补充。

百科界面要求：

- 左侧区域以列表形式展示条目合集与下属条目。
- 右侧区域呈现当前选中条目的详细内容。
- 初始状态下右侧内容为空。
- 内容高度超出显示区域时，应支持垂直滚动；若实现滚动困难，可拆成分页。

## 教程

教程建议采用脚本化步骤：

- 提示文本。
- 高亮 UI 节点。
- 等待玩家操作。
- 限制可点击区域。
- 完成后进入下一步。

这样便于把教学关和正常战斗复用同一套 UI。

## 成长/奖励界面

关卡胜利后显示：

- 角色头像。
- 属性变化。
- 新技能或技能升级。
- 继续按钮。

建议这个界面只读取 `growth_rewards` 配置。
