# 战斗系统

## 时间轴

1. 玩家与敌方在此处轮流放置指令，规划当前回合行动

2. 时间轴分为六个时点，每个时点只能容纳一个指令

3. 每回合分为准备阶段与演出阶段。在准备阶段，玩家将为己方角色在时间轴上放置指令。在演出阶段，指令将依据其在时间轴上的顺序与优先级依次执行


## 准备阶段

- 准备阶段开始时，显示“第N回合”（N为当前回合数）同时有对应音效，需要通过程序读取对应回合，显示正确的回合数。1.5秒后显示“准备阶段“，同时有对应音效。

- 顺序为怪物优先。时间轴右侧显示怪物“还剩X次”。怪物的准备阶段开始时，每个X≠0的怪物在时间轴上放置一次行动。放置的逻辑详见怪物设计部分。计算怪物行动时，时间轴右侧显示“思考中”。怪物的行动放置同时完成并有音效提示。完成后，显示新的“还剩X次”，若X=0，显示“完成”。

- 这之后，玩家开始准备。于左侧显示战斗UI交互控件。详见战斗UI交互控件。

- 玩家对拥有剩余行动次数的每个角色安排一次行动，于时间轴对应位置长按以确定，确定后有音效提示，确定后不可撤回。可行动的角色全部安排完成后自动进入怪物的下一轮准备。

- 场上无可行动角色后，立刻揭示所有怪物的行动（对应时点显示技能优先级与技能名称），在后台开始时间轴的结算，并在左侧出现“开始结算”按钮。该按钮玩家点击后进入演出阶段，应当在这之前完成时间轴的结算（该设计目的是为时间轴的结算拖一些时间，并优化游戏手感）

# 准备阶段 UI

## 战斗UI交互控件

- 显示在屏幕左侧，我方时间轴的上方区域。只在准备阶段且我方行动时显示出。
- 包含“设置指令”、“检视详情”、“开始结算”

### 设置指令

1. 点击“设置指令”，进入次级菜单，显示我方角色。其中战败或行动次数为0的显示灰色且不可选区。
2. 选择具体角色后，从上往下依次出现“近战”“位移”“远程”“特殊”（若该角色无某一类型技能则不显示）。
3. 点击后展示拥有对应标签的技能。
4. 用户鼠标置于技能上时，对应技能字体放大，并在技能列表下方展示技能详情，包括优先级、MP消耗与技能效果介绍。
5. 单击技能后，于时间轴上空白节点长按1.2秒以放置技能。需要做一个随长按时间增长的进度条效果。并在我方时间轴上方“我方行动”的上方显示“于时间轴上空白节点长按以放置技能”
6. 在选择 以角色为目标的技能 后，应该展示一个次级列表，包含所有敌人名称，其中已战败的敌人显示为灰色，不可选取。

### 检视详情

点击后，鼠标置于非空白时点上，在右侧界面显示该技能的描述。若未被揭示，仅显示其标签。若已被揭示，显示其优先级、描述、目标区域/角色（若该技能需要指定）。

### 开始结算

当场上所有单位均无可行动机会，将“设置指令”替换为“开始结算”。

点击后弹窗“是否进入演出阶段”，点击“是”进入演出阶段，点击“否”取消弹窗。

# 技能说明

**所有技能结算时都要遵循以下基本逻辑**

以目标为对象的近战攻击，若无特殊说明，必须检测是否与目标在同区域。同区域，正常结算。不同区域，战斗详情显示“失败-与目标不在同一区域”，不产生任何结果，结算下一个指令。

以目标为对象的移动技能（包括冲锋），若无特殊说明，必须检测是否与目标在同区域。不同区域，正常结算。同区域，战斗详情显示“失败-与目标在同一区域”，不产生任何结果，结算下一个指令。（和近战攻击相反）

以区域为对象的移动技能，若自己所在区域与目标区域一致，视为无法执行。战斗详情显示“失败-已处于目标区域”，不产生任何结果，结算下一个指令。

以目标为对象的任何技能，若结算时，目标生命值为0，则战斗详情显示“失败-目标已失去战斗能力”，不产生任何结果，结算下一个指令。

一定时点后生效的范围伤害技能

均在对应时点所有结算完成后，才结算该范围伤害。（可以设计为赋予时点特殊状态，在该时点结束时触发）（举例：在时点1释放4个时点后生效的范围伤害技能，则时点5所有结算后完成后，结算该伤害。）

若本回合结束时未触发，可以延续到下回合。（举例：在时点4释放4个时点后生效的范围伤害技能，则下回合时点2触发）

技能需要设置标签，以便于状态效果/技能触发等的检索：近战/远程/移动/防御、单体/范围、治疗

注意：远程的延迟触发与单位是否存活无关，相当于技能释放后，有一个独立的“技能”判定，会在对应时点作为优先级为1（即最低优先级，若有多个延迟判定同时触发，则玩家优先级大于怪物）的事件触发。 而近战的2个时点是持续2个时点，在放置的时点是无事发生的，在下一个时点才进行结算。整个流程是近战的同一个技能，优先级不变，并且若释放者战败会无法结算该技能，结算时按照即时区域结算，而非释放时所在区域。（即，若释放者在结算前被移动至其他区域，则结算时是在新区域结算）

# 关卡与怪物设计

敌方的技能释放的基本逻辑：

1. 在判定是否有同区域玩家角色时，不考虑玩家指令带来的影响。

2. 每个时点都设置“同区域是否有玩家角色的判定结果”。回合开始时根据玩家与敌方的站位分布，更新每个时点上的“同区域是否有玩家角色的判定结果”；当敌方放置包含位移效果的技能指令后，更新后续时点的“同区域是否有玩家角色的判定结果”。

3. “同区域是否有玩家角色的判定结果”为是时，追加判定“同区域玩家角色数量”。

4. 释放时，**若有多个可选技能，则从可释放的技能中随机选取**。（不同于技能释放目标的优先级。目前只有“狂怒”这个技能做了释放最优先级）

# 百科

## UI位置

位于右上角，书本类外观。（新手教程时建议给一个打开百科的说明）

## 界面

在百科界面中，左侧区域以列表形式展示各条目合集与下属条目，右侧区域用于呈现当前选中条目的详细内容。

初始状态下，右侧内容区域为空。

用户点击左侧任意条目后，右侧动态加载并展示对应的内容。当内容高度超出显示区域时，采用垂直滚动条进行浏览。若滚动条实现存在技术限制，则将该条目内容拆分为多个逻辑段落，并通过分页控件（如“上一页”/“下一页”或页码导航）进行分段展示。
